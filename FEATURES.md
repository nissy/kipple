# Kipple Feature Guide

Kipple は、macOS のメニューバーから使えるテキスト向けクリップボードマネージャーです。
コピーしたテキストを自動で記録し、あとから検索、再コピー、編集、分類、順番貼り付けできます。

データは Mac の中に保存されます。クリップボード履歴、設定、カテゴリ情報は外部サービスへ送信されません。

## 主な特徴

- コピーしたテキストを自動で履歴に保存
- 履歴を検索、ピン留め、カテゴリで整理
- コピー元アプリとウィンドウ名を表示
- 履歴を選んでクリップボードへ戻す
- Live Editor で内容を整えてから保存
- JSON / YAML の整形に対応
- 複数の履歴を順番に貼り付ける Queue
- 画面上の文字を範囲選択して OCR で取り込み
- 日本語 / 英語 UI に対応

## クリップボード履歴

Kipple はテキストのコピーを監視し、履歴として保存します。
履歴は新しい順に表示され、同じ内容をもう一度コピーした場合は重複を増やさず、既存の項目を先頭へ移動します。

履歴には次の情報が含まれます。

- コピーしたテキスト
- コピー日時
- コピー元アプリ
- コピー元ウィンドウ名
- URL か通常テキストかの分類
- ピン留め状態
- ユーザーが割り当てたカテゴリ

履歴行にマウスを重ねると、内容のプレビュー、文字数、コピー日時、コピー元情報を確認できます。
長いテキストはプレビューで先頭部分を確認できます。

## 検索とフィルタ

履歴上部の検索欄から、コピーした内容をすばやく絞り込めます。
検索対象には本文とコピー元アプリ名が含まれます。

フィルタでは次の絞り込みができます。

- ピン留め項目のみ表示
- URL のみ表示
- ユーザー定義カテゴリで表示
- None カテゴリで表示

カテゴリフィルタは、カテゴリ管理画面で表示 / 非表示を切り替えられます。

## ピン留め

よく使う履歴はピン留めできます。
ピン留めした項目は履歴削除の対象から外れ、重要なテキストを残しておけます。

ピン留め数には上限があります。
初期値は 50 件で、設定から 1〜100 件の範囲で変更できます。

## カテゴリ

Kipple には組み込みカテゴリとして `None` と `URL` があります。
URL らしいテキストは自動的に URL として分類されます。

ユーザーは独自カテゴリを追加できます。
カテゴリには名前とアイコンを設定でき、履歴行のカテゴリメニューから各項目へ割り当てられます。

カテゴリ管理では次の操作ができます。

- カテゴリの追加
- 名前の変更
- アイコンの変更
- フィルタへの表示 / 非表示
- ユーザー定義カテゴリの削除

カテゴリを削除した場合、そのカテゴリが付いた履歴項目は `None` に移されます。
組み込みカテゴリは削除できません。

## URL を開く

URL や URI として開ける履歴項目は、設定した修飾キーを押しながらクリックすると開けます。
コンテキストメニューから開くこともできます。

対応する代表例は次の通りです。

- `http`
- `https`
- `ftp`
- `mailto`
- アプリ側でハンドラが登録されている URI スキーム

通常クリックでは、項目をクリップボードへ戻します。
誤操作を防ぐため、URL を開く操作には修飾キーが必要です。

## Live Editor

Kipple には、現在のクリップボード内容を確認・編集するエディタがあります。

エディタでは次の操作ができます。

- 現在のクリップボード内容を表示
- ダブルクリックまたは編集ボタンで編集開始
- 編集内容を履歴に保存
- 内容をクリア
- 前後の空白や改行をトリム
- JSON を読みやすく整形
- YAML を読みやすく整形

エディタは行番号付きです。

編集内容を履歴に保存すると、コピー元は Kipple の Live Editor として記録されます。
同じ内容を保存した場合は、通常の履歴と同じく重複を増やさず先頭へ移動します。

## 行分割保存

履歴項目のコンテキストメニューから、複数行のテキストを行ごとに分割して履歴へ追加できます。
空行は除外され、各行は前後の空白を取り除いて保存されます。

住所、リスト、フォーム入力用の値などをまとめてコピーしたあと、個別の履歴項目として使いたい場合に便利です。

## Queue

Queue は、複数の履歴項目を指定した順番で貼り付ける機能です。
フォーム入力や定型作業のように、複数の値を順番に貼り付けたい場面で使えます。

Queue を有効にすると、履歴項目をクリックしてキューに追加できます。
追加された項目には順番の番号が表示されます。

主な操作は次の通りです。

- Queue ボタンで Queue モードを開始 / 終了
- クリックで項目を追加または解除
- Shift + クリックで範囲選択
- Queue 中は次に貼り付ける項目がクリップボードに入る
- Command + V を押すたびに次の項目へ進む
- Loop を有効にすると、最後まで進んだあと先頭へ戻る

Queue は Command + V を検知するため、アクセシビリティ権限が必要です。
権限がない場合、Queue ボタンには警告が表示され、設定の Permission タブから許可できます。

## Paste on Selection

Paste on Selection を有効にすると、履歴項目を選んだときにクリップボードへ戻すだけでなく、前面アプリへ自動貼り付けできます。

この機能もキーボード操作の自動化を使うため、アクセシビリティ権限が必要です。
権限がない場合は設定項目が無効になります。

## 画面テキストキャプチャ

Screen Text Capture は、画面上の文字を範囲選択して OCR で取り込む機能です。
選択した範囲の画像を一時的に取得し、macOS の Vision フレームワークでテキストを認識します。

認識したテキストは次のように扱われます。

- 前後の空白を取り除く
- クリップボードへコピー
- 履歴へ保存
- Kipple のメインウィンドウを開く

認識言語は日本語と英語に対応しています。
画面を読み取るため、画面収録権限が必要です。
取得した画像と認識テキストは端末内で処理され、外部へ送信されません。

## ホットキー

設定から次のホットキーを変更できます。

- Kipple を開くグローバルホットキー
- エディタ内容を履歴に保存するホットキー
- エディタをクリアするホットキー
- 画面テキストキャプチャのホットキー

ホットキーが他のアプリや macOS の設定と競合している場合は、別の組み合わせを選んでください。

## 設定

設定画面は4つのタブに分かれています。

### General

- 表示言語
- ログイン時に起動
- Kipple を開くグローバルホットキー
- ウィンドウアニメーション

### Editor

- エディタ位置
- 保存ホットキー
- クリアホットキー

### Clipboard

- URL を開く修飾クリック
- カテゴリ管理
- Auto Pin
- 最大履歴数
- 最大ピン留め数
- 履歴削除
- 画面テキストキャプチャ
- Paste on Selection

### Permission

- 画面収録権限の状態確認
- アクセシビリティ権限の状態確認
- 権限が必要な理由
- macOS の System Settings への案内

## Auto Pin

Auto Pin を有効にすると、同じテキストを短時間に繰り返しコピーしたとき、自動的にピン留めできます。

設定できる項目は次の通りです。

- 有効 / 無効
- 繰り返しコピーとみなす時間
- ピン留めに必要なコピー回数

履歴からの再コピーは Auto Pin の判定に含まれません。
外部アプリで繰り返しコピーした内容が対象です。

## データ管理

履歴の最大件数は設定から変更できます。
初期値は 300 件で、10〜1000 件の範囲で設定できます。

履歴をまとめて削除することもできます。
この操作では、ピン留めされていない履歴だけを削除し、ピン留め項目は残します。

保存対象はテキストです。
画像やファイルそのものを履歴として保存する機能はありません。

## プライバシー

Kipple はローカルで動作するアプリです。

- クリップボード履歴は Mac の中に保存
- 設定は UserDefaults に保存
- カテゴリは UserDefaults に保存
- 履歴は SwiftData に保存
- OCR は端末内で処理
- 分析、トラッキング、クラウド同期は使用しない

クリップボードには機密情報が含まれることがあります。
不要になった履歴は設定画面から削除できます。

## 必要な権限

### 画面収録

Screen Text Capture で、ユーザーが選択した画面領域を読み取るために必要です。
この権限がない場合、画面テキストキャプチャは使えません。

### アクセシビリティ

Queue と Paste on Selection で、Command + V の検知や貼り付け操作を扱うために必要です。
この権限がない場合、該当機能は無効になります。

## 動作環境

- macOS 14.0 以降
- Apple Silicon / Intel Mac

## できないこと

現在の Kipple は、テキスト中心のクリップボードマネージャーです。

- 画像履歴の保存
- ファイル履歴の保存
- クラウド同期
- チーム共有
- Web サービスへの履歴送信

これらの機能は現在の実装には含まれていません。
